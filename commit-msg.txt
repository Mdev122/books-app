feat: books-app - project scaffold, rendering, favorites and filters

Struktura projektu:
- src/index.html, src/js/, src/sass/, src/css/, src/images/, src/vendor/
  zgodnie z npm scripts z package.json (init:dirs / init:files)

data.js:
- dataSource.books - lista książek z polami id, name, price, rating, image
- details.adults / details.nonFiction do obsługi filtrowania

functions.js:
- utils.createDOMFromHTML - generuje element DOM na podstawie stringa HTML
  zwróconego przez skompilowany szablon Handlebars
- Handlebars helpers: ifEquals, joinValues

script.js:
- Ćwiczenie 1: render() - iteruje po dataSource.books, kompiluje szablon
  #template-book i dokleja wygenerowane elementy do .books-list
- Ćwiczenie 2: dodawanie książek do ulubionych przez dblclick na okładce
  (.book__image), zapis id w tablicy favoriteBooks + klasa .favorite
- Ćwiczenie 3: toggle ulubionych - ponowny dblclick usuwa z favoriteBooks
  i zdejmuje klasę .favorite
- Ćwiczenie 4: refaktoryzacja na event delegation - jeden nasłuchiwacz
  dblclick na .books-list zamiast osobnego per książka, z użyciem
  event.target.closest('.book__image')
- Ćwiczenie 5: filtrowanie po formularzu .filters (event delegation) -
  tablica filters aktualizowana przez push/splice przy zmianie checkboxów,
  filterBooks() dodaje/usuwa klasę .hidden dla książek niepasujących
  do aktywnych filtrów (details.adults / details.nonFiction)

Style (style.scss / style.css):
- reset .books-list (list-style, padding, margin)
- .book__image.favorite - złota ramka (outline)
- .book__image.hidden - grayscale + opacity dla odfiltrowanych książek

Fix:
- handlebars.js ładowany z CDN (cdnjs 4.7.8) zamiast nieistniejącego
  lokalnego pliku vendor/handlebars.min.js
